# 🌐 06 - 第三方登录实现

> "使用微信登录"、"使用 GitHub 登录"、"使用 Google 登录"—— 这些我们每天都在使用的功能，底层是如何工作的？本章将详细介绍第三方登录的实现原理和流程。

---

## 一、什么是第三方登录？

### 1.1 通俗理解

第三方登录就像是用**身份证**在酒店办理入住：

- 你不需要在酒店重新办一个"酒店身份证"（注册新账号）
- 你出示公安局颁发的身份证（已有的微信/GitHub 账号）
- 酒店确认身份证是真的（通过 OAuth 验证）
- 你就可以入住了（登录成功）

### 1.2 第三方登录的优势

| 对于用户 | 对于开发者 |
|----------|------------|
| ✅ 无需注册新账号 | ✅ 降低注册门槛，提高转化率 |
| ✅ 无需记住新密码 | ✅ 不用自己管理密码（减少安全负担） |
| ✅ 一键登录，体验好 | ✅ 可获取用户社交信息（头像、昵称） |
| ✅ 信任大平台的安全性 | ✅ 利用大平台的安全能力 |

## 二、第三方登录的通用架构

所有第三方登录都基于 **OAuth 2.0 授权码模式**，流程基本一致：

```mermaid
sequenceDiagram
    participant 用户 as 👤 用户
    participant 你的网站 as 🌐 你的网站
    participant 第三方 as 🏢 第三方平台<br/>（微信/GitHub/Google）

    rect rgb(232, 245, 233)
        Note over 用户,第三方: 阶段1：用户授权
        用户->>你的网站: ① 点击"使用XX登录"
        你的网站-->>用户: ② 重定向到第三方授权页
        用户->>第三方: ③ 用户在第三方平台上登录并同意授权
        第三方-->>用户: ④ 重定向回你的网站，携带授权码 code
    end

    rect rgb(227, 242, 253)
        Note over 你的网站,第三方: 阶段2：获取Token（后端）
        用户->>你的网站: ⑤ 浏览器带着 code 访问回调地址
        你的网站->>第三方: ⑥ 用 code + client_secret 换取 access_token
        第三方-->>你的网站: ⑦ 返回 access_token
    end

    rect rgb(243, 229, 245)
        Note over 你的网站,第三方: 阶段3：获取用户信息
        你的网站->>第三方: ⑧ 用 access_token 调用用户信息API
        第三方-->>你的网站: ⑨ 返回用户信息（openid、昵称、头像等）
    end

    rect rgb(255, 243, 224)
        Note over 用户,你的网站: 阶段4：完成登录
        你的网站->>你的网站: ⑩ 查找或创建本地用户账号
        你的网站->>你的网站: ⑪ 创建本地登录Session/Token
        你的网站-->>用户: ⑫ 登录成功！
    end
```

## 三、GitHub 登录实现

GitHub 是最容易上手的第三方登录平台之一。

### 3.1 准备工作

```mermaid
graph TD
    A["第一步：注册 OAuth App"] --> B["GitHub Settings → Developer Settings → OAuth Apps → New"]
    B --> C["填写信息"]
    C --> C1["Application name: 你的应用名"]
    C --> C2["Homepage URL: https://yoursite.com"]
    C --> C3["Authorization callback URL: https://yoursite.com/auth/github/callback"]
    C --> D["获得 Client ID 和 Client Secret"]
    
    style D fill:#c8e6c9
```

### 3.2 完整流程

```mermaid
sequenceDiagram
    participant 用户 as 👤 用户
    participant 前端 as 🌐 前端
    participant 后端 as 🖥️ 后端服务器
    participant GitHub as 🐙 GitHub

    用户->>前端: 点击"GitHub登录"
    前端->>用户: 重定向到 GitHub
    Note over 前端: GET https://github.com/login/oauth/authorize<br/>?client_id=YOUR_CLIENT_ID<br/>&redirect_uri=https://yoursite.com/auth/github/callback<br/>&scope=read:user user:email<br/>&state=random_string

    用户->>GitHub: 在GitHub页面登录并授权
    GitHub-->>用户: 302 重定向到回调URL
    Note over GitHub: Location: https://yoursite.com/auth/github/callback<br/>?code=AUTHORIZATION_CODE&state=random_string

    用户->>后端: 浏览器访问回调URL（携带code）
    
    后端->>后端: 验证 state 参数（防CSRF）
    后端->>GitHub: POST https://github.com/login/oauth/access_token<br/>{client_id, client_secret, code}
    GitHub-->>后端: {access_token: "gho_xxxx", token_type: "bearer"}

    后端->>GitHub: GET https://api.github.com/user<br/>Authorization: Bearer gho_xxxx
    GitHub-->>后端: {id: 12345, login: "zhangsan", avatar_url: "...", email: "..."}

    后端->>后端: 查找本地用户（根据github_id）<br/>不存在则自动创建
    后端->>后端: 签发本地JWT / 创建Session
    后端-->>用户: 登录成功 + 设置Cookie/返回Token
```

### 3.3 代码示例（Node.js）

```javascript
const express = require('express');
const axios = require('axios');
const jwt = require('jsonwebtoken');

const app = express();

const GITHUB_CLIENT_ID = 'your_client_id';
const GITHUB_CLIENT_SECRET = 'your_client_secret';
const REDIRECT_URI = 'https://yoursite.com/auth/github/callback';

// 步骤1：引导用户到 GitHub 授权页面
app.get('/auth/github', (req, res) => {
  const state = generateRandomString(32); // 生成随机state防CSRF
  // 存储 state 到 session 以便后续验证
  req.session.oauthState = state;
  
  const githubAuthUrl = `https://github.com/login/oauth/authorize` +
    `?client_id=${GITHUB_CLIENT_ID}` +
    `&redirect_uri=${encodeURIComponent(REDIRECT_URI)}` +
    `&scope=read:user user:email` +
    `&state=${state}`;
  
  res.redirect(githubAuthUrl);
});

// 步骤2：处理 GitHub 回调
app.get('/auth/github/callback', async (req, res) => {
  const { code, state } = req.query;
  
  // 验证 state 防 CSRF
  if (state !== req.session.oauthState) {
    return res.status(403).json({ error: 'State不匹配，可能遭受CSRF攻击' });
  }
  
  try {
    // 步骤3：用 code 换取 access_token
    const tokenResponse = await axios.post(
      'https://github.com/login/oauth/access_token',
      {
        client_id: GITHUB_CLIENT_ID,
        client_secret: GITHUB_CLIENT_SECRET,
        code: code,
        redirect_uri: REDIRECT_URI
      },
      { headers: { Accept: 'application/json' } }
    );
    
    const accessToken = tokenResponse.data.access_token;
    
    // 步骤4：用 access_token 获取用户信息
    const userResponse = await axios.get('https://api.github.com/user', {
      headers: { Authorization: `Bearer ${accessToken}` }
    });
    
    const githubUser = userResponse.data;
    // githubUser = { id: 12345, login: 'zhangsan', avatar_url: '...', email: '...' }
    
    // 步骤5：查找或创建本地用户
    let localUser = await db.findUserByGithubId(githubUser.id);
    if (!localUser) {
      localUser = await db.createUser({
        github_id: githubUser.id,
        username: githubUser.login,
        avatar: githubUser.avatar_url,
        email: githubUser.email
      });
    }
    
    // 步骤6：签发本地 JWT
    const token = jwt.sign(
      { userId: localUser.id, username: localUser.username },
      'your-secret-key',
      { expiresIn: '7d' }
    );
    
    // 重定向到前端，并携带 token
    res.redirect(`https://yoursite.com/login/success?token=${token}`);
    
  } catch (error) {
    res.status(500).json({ error: '登录失败' });
  }
});
```

## 四、微信登录实现

微信登录是国内最常用的第三方登录方式，分为"网站应用微信登录"和"公众号/小程序登录"两种。

### 4.1 微信网站应用登录流程

```mermaid
sequenceDiagram
    participant 用户 as 👤 用户
    participant 网站 as 🌐 你的网站
    participant 微信 as 💚 微信开放平台

    用户->>网站: ① 点击"微信登录"
    网站-->>用户: ② 重定向到微信授权页（显示二维码）
    Note over 网站: https://open.weixin.qq.com/connect/qrconnect<br/>?appid=APPID<br/>&redirect_uri=REDIRECT_URI<br/>&response_type=code<br/>&scope=snsapi_login<br/>&state=STATE

    用户->>用户: ③ 用手机微信扫描二维码
    用户->>微信: ④ 在手机上确认授权
    微信-->>用户: ⑤ 重定向回网站 + code
    
    用户->>网站: ⑥ 浏览器携带 code 访问回调地址
    网站->>微信: ⑦ POST 用 code + appid + secret 换 access_token
    微信-->>网站: ⑧ 返回 {access_token, openid, unionid}
    
    网站->>微信: ⑨ GET 用 access_token + openid 获取用户信息
    微信-->>网站: ⑩ 返回 {nickname, headimgurl, sex, ...}
    
    网站->>网站: ⑪ 关联/创建本地账号
    网站-->>用户: ⑫ 登录成功
```

### 4.2 微信特殊概念

```mermaid
graph TD
    A["微信用户标识体系"] --> B["OpenID"]
    A --> C["UnionID"]
    
    B --> B1["同一个用户在不同应用下<br/>OpenID 是不同的<br/>"]
    B --> B2["App A 中: oA123<br/>App B 中: oB456<br/>公众号中: oC789"]
    
    C --> C1["同一个用户在同一开放平台下<br/>所有应用的 UnionID 是相同的"]
    C --> C2["App A 中: u-same-001<br/>App B 中: u-same-001<br/>公众号中: u-same-001"]
    
    D["💡 如果你有多个微信应用<br/>（APP + 公众号 + 小程序）<br/>必须使用 UnionID 来关联同一用户"]

    style C fill:#c8e6c9
    style D fill:#fff3e0
```

| 标识 | 说明 | 唯一范围 | 使用场景 |
|------|------|----------|----------|
| **OpenID** | 用户在某个应用下的唯一标识 | 单个应用 | 只有一个微信应用时 |
| **UnionID** | 用户在开放平台下的唯一标识 | 同一开放平台账号下的所有应用 | 多个微信应用时（推荐） |

## 五、Google 登录实现

Google 登录使用 **OpenID Connect（OIDC）** 协议，在 OAuth 2.0 基础上增加了 ID Token。

### 5.1 Google 登录流程

```mermaid
sequenceDiagram
    participant 用户 as 👤 用户
    participant 网站 as 🌐 你的网站
    participant Google as 🔵 Google

    用户->>网站: 点击"Google登录"
    网站-->>用户: 重定向到 Google 授权页
    Note over 网站: https://accounts.google.com/o/oauth2/v2/auth<br/>?client_id=CLIENT_ID<br/>&redirect_uri=REDIRECT_URI<br/>&response_type=code<br/>&scope=openid email profile<br/>&state=STATE

    用户->>Google: 选择Google账号并授权
    Google-->>用户: 重定向回网站 + code

    网站->>Google: 用 code 换取 Token
    Google-->>网站: {access_token, id_token, refresh_token}
    
    Note over 网站: id_token 是 JWT 格式<br/>直接解析即可获取用户信息：<br/>{sub: "12345", name: "Zhang San",<br/> email: "zhangsan@gmail.com",<br/> picture: "https://..."}
    
    网站->>网站: 解析 id_token 获取用户信息
    网站->>网站: 关联/创建本地账号
    网站-->>用户: 登录成功
```

> 💡 **Google 的优势**：返回的 `id_token` 是 JWT 格式，直接解码就能拿到用户信息，不需要再额外调 API。

## 六、账号关联策略

当用户使用第三方登录时，需要将第三方账号与本地账号关联。这里有几种常见策略：

### 6.1 数据库设计

```mermaid
erDiagram
    USER {
        int id PK
        string username
        string email
        string password_hash
        string avatar
        datetime created_at
    }
    
    USER_OAUTH {
        int id PK
        int user_id FK
        string provider "github/wechat/google"
        string provider_user_id "第三方平台的用户ID"
        string access_token
        string refresh_token
        datetime token_expires_at
        datetime created_at
    }
    
    USER ||--o{ USER_OAUTH : "一个用户可绑定多个第三方"
```

### 6.2 账号关联的场景

```mermaid
graph TD
    A["用户使用第三方登录"] --> B{"根据 provider + provider_user_id<br/>查找 user_oauth 表"}
    
    B -->|找到记录| C["已绑定过 ✅<br/>直接登录关联的本地账号"]
    
    B -->|未找到| D{"根据第三方邮箱<br/>查找 user 表"}
    D -->|找到同邮箱用户| E["提示：该邮箱已注册<br/>请先登录已有账号再绑定"]
    D -->|未找到| F["自动创建新用户<br/>+ 创建 user_oauth 绑定记录"]
    
    G["已登录用户手动绑定"] --> H["在个人设置中点击'绑定GitHub'"]
    H --> I["走 OAuth 流程获取第三方信息"]
    I --> J["创建 user_oauth 记录<br/>关联到当前登录的用户"]

    style C fill:#c8e6c9
    style F fill:#c8e6c9
    style J fill:#c8e6c9
    style E fill:#fff3e0
```

### 6.3 关联策略总结

| 场景 | 处理方式 |
|------|----------|
| 第三方 ID 已绑定 | 直接登录关联的本地账号 |
| 第三方 ID 未绑定，邮箱匹配到已有用户 | 提示用户先登录再绑定（或自动关联） |
| 第三方 ID 未绑定，邮箱也没匹配到 | 自动创建新用户 + 绑定 |
| 已登录用户手动绑定 | 创建 user_oauth 记录 |
| 用户解绑第三方账号 | 删除 user_oauth 记录（确保至少保留一种登录方式） |
