# 第七册：附录：术语表与训练命令速查

> 这一册定位是“速查手册”。
>
> 当前面几册讲的是原理、方法、排障、运维，这一册则负责把最常用的术语、命令、目录规范和交接模板收束起来，方便你在实战中快速翻查。

---

## 1. 核心术语表

| 术语 | 英文 | 解释 |
|------|------|------|
| 大模型 | LLM | 大规模语言模型 |
| 预训练 | Pre-Training | 用海量语料训练基座能力 |
| 后训练 | Post-Training | 在已有模型上做任务适配与对齐 |
| 监督微调 | SFT | 用标准输入输出样本训练 |
| 继续预训练 | CPT | 用领域语料继续灌入知识 |
| 参数高效微调 | PEFT | 只训练少量附加参数 |
| 低秩适配 | LoRA | 最常用的 PEFT 方法 |
| 量化 LoRA | QLoRA | 低比特加载基座 + LoRA 训练 |
| 偏好优化 | Preference Optimization | 根据偏好信号优化回答质量 |
| 奖励模型 | RM | 给回答打分的模型 |
| 参考模型 | Reference Model | 训练时的行为参照 |
| 损失 | Loss | 预测与目标的差距 |
| 梯度 | Gradient | 参数更新方向 |
| 学习率 | Learning Rate | 每次参数更新幅度 |
| 检查点 | Checkpoint | 训练中间保存的快照 |
| 灾难性遗忘 | Catastrophic Forgetting | 新任务训练导致旧能力退化 |
| 幻觉 | Hallucination | 模型自信但不真实地编造内容 |

## 2. 常用数据格式速查

### 2.1 SFT 单轮格式

```json
{"instruction": "请把下面文本总结成一句话", "input": "......", "output": "......"}
```

### 2.2 SFT 多轮格式

```json
{
  "messages": [
    {"role": "system", "content": "你是一个有帮助的助手"},
    {"role": "user", "content": "什么是后训练？"},
    {"role": "assistant", "content": "后训练是在已有模型上继续做任务适配和对齐。"}
  ]
}
```

### 2.3 DPO 偏好对格式

```json
{
  "messages": [
    {"role": "user", "content": "如何提高学习效率？"}
  ],
  "chosen": {"role": "assistant", "content": "可以从明确目标、拆分任务、固定时间块和减少打断四方面入手。"},
  "rejected": {"role": "assistant", "content": "认真一点就行。"}
}
```

### 2.4 CPT 文本格式

```json
{"text": "这里是一段领域文本，用于继续预训练。"}
```

## 3. 常见训练命令速查

下面用的是“示意性命令”，核心是帮助你理解训练任务到底由哪些配置构成。

### 3.1 `ms-swift` 的 SFT LoRA 示例

```bash
swift sft \
  --model /models/qwen2.5-7b-instruct \
  --dataset /data/train.jsonl \
  --train_type lora \
  --output_dir /output/run-001 \
  --num_train_epochs 3 \
  --learning_rate 1e-4 \
  --per_device_train_batch_size 4 \
  --gradient_accumulation_steps 4 \
  --max_length 4096 \
  --lora_rank 64 \
  --lora_alpha 128 \
  --torch_dtype bf16
```

### 3.2 `ms-swift` 的 DPO 示例

```bash
swift rlhf \
  --rlhf_type dpo \
  --model /models/qwen2.5-7b-instruct \
  --dataset /data/dpo.jsonl \
  --train_type lora \
  --output_dir /output/dpo-run-001 \
  --num_train_epochs 2 \
  --learning_rate 5e-6 \
  --per_device_train_batch_size 2 \
  --gradient_accumulation_steps 8 \
  --max_length 4096 \
  --lora_rank 64 \
  --torch_dtype bf16
```

### 3.3 `LLaMA-Factory` 的 SFT LoRA 示例

```bash
llamafactory-cli train \
  --stage sft \
  --do_train true \
  --model_name_or_path /models/qwen2.5-7b-instruct \
  --dataset /data/train.jsonl \
  --finetuning_type lora \
  --output_dir /output/run-001 \
  --learning_rate 1e-4 \
  --num_train_epochs 3 \
  --cutoff_len 4096 \
  --per_device_train_batch_size 4 \
  --gradient_accumulation_steps 4 \
  --bf16 true
```

## 4. 训练输出目录建议

建议每次训练都采用清晰目录结构：

```text
train_runs/
  └─ search-qwen2.5-7b-lora-v20260324-001/
      ├─ config/
      │   ├─ train.yaml
      │   └─ metadata.json
      ├─ logs/
      │   ├─ train.log
      │   └─ events.out.tfevents...
      ├─ checkpoints/
      │   ├─ checkpoint-500/
      │   ├─ checkpoint-1000/
      │   └─ final/
      ├─ eval/
      │   ├─ offline_eval.json
      │   └─ human_review.md
      └─ release/
          └─ release_note.md
```

这样做的好处是：

- 排障更快
- 交接更清楚
- 回滚更容易

## 6. 一份值班速查清单

如果你是值班同学，训练任务出问题时优先确认：

1. 任务状态是否还在运行
2. 最近一条日志是什么
3. GPU 是否 OOM
4. checkpoint 是否还在写
5. 节点是否掉线
6. 输出目录是否写满
7. 是否为某次镜像或代码升级后集中出现

---

## 7. 一份最小训练闭环模板

如果你是第一次做后训练，建议只跑这套最小闭环：

1. 选一个成熟开源 Instruct 模型
2. 准备 200 到 1000 条高质量 SFT 样本
3. 用 LoRA 跑一轮
4. 建一个 100 条左右的验证集
5. 建一个 100 条左右的测试集
6. 跑离线评测
7. 做人工抽检
8. 写一份训练交接单

这套闭环跑通后，再考虑更复杂的 DPO、GRPO 或生产化流程。
